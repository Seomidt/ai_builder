  id uuid primary key default gen_random_uuid(),
  tenant_id uuid,
  user_id uuid,
  feature text not null,
  model text not null,
  prompt_tokens integer not null default 0,
  completion_tokens integer not null default 0,
  total_tokens integer not null default 0,
  input_preview text,
  status text not null default 'success',
  error_message text,
  latency_ms integer,
  created_at timestamptz not null default now()

create index ai_usage_user_id_idx on public.ai_usage (user_id);
create index ai_usage_feature_idx on public.ai_usage (feature);
create index ai_usage_created_at_idx on public.ai_usage (created_at desc);

TASK 2 — CREATE SERVER-ONLY AI MODULES
==================================================

Create these files if they do not already exist:

- lib/ai/client.ts
- lib/ai/config.ts
- lib/ai/service.ts
- lib/ai/usage.ts

Implementation requirements:

A) lib/ai/client.ts
- Must be server-only
- Use import "server-only"
- Initialize and export an OpenAI client using process.env.OPENAI_API_KEY
- Do not add any client-side usage
- Keep this file minimal and focused

B) lib/ai/config.ts
- Centralize AI configuration
- Export:

AI_MODELS = {
  default: "gpt-4.1-mini"
} as const

AI_TIMEOUT_MS = 20000

- Keep this as the central place for model defaults and timeout config
- Do not hardcode model names elsewhere if avoidable

C) lib/ai/service.ts
- Must be server-only
- Use import "server-only"
- Implement a reusable function named generateText()

Function requirements:
- Typed input
- Typed output
- Accept:

{
  systemPrompt: string
  userInput: string
  model?: keyof typeof AI_MODELS
}

Implementation requirements:
- Use the OpenAI Responses API
- Use the configured model from AI_MODELS
- Return an object shaped like:

{
  text: string
  usage: {
    input_tokens: number
    output_tokens: number
    total_tokens: number
  } | null
  raw: <raw OpenAI response>
}

- Keep code clean and production-oriented
- Avoid feature-specific assumptions
- Do not mix logging into this module unless clearly necessary
- Keep responsibilities separated

D) lib/ai/usage.ts
- Must be server-only
- Use import "server-only"
- Implement a reusable function named logAiUsage()

Function requirements:
- Insert rows into public.ai_usage
- Use existing Supabase admin/service-role helper utilities if the project already has them
- If the project does not already have a server/admin Supabase helper, create the minimum consistent server-side pattern required, but do not refactor unrelated code

Suggested payload shape for logAiUsage:
{
  tenantId?: string | null
  userId?: string | null
  feature: string
  model: string
  promptTokens?: number
  completionTokens?: number
  totalTokens?: number
  inputPreview?: string | null
  status: "success" | "error"
  errorMessage?: string | null
  latencyMs?: number | null
}

Behavior requirements:
- Logging failures must not crash application flow
- If insert fails, only console.error
- Do not expose service-role logic outside server code
